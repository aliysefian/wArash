Setting Up a Local WordPress Site with Apache, MySQL, and Git This guide explains how to install Apache, clone a WordPress site from a Git repository, install MySQL, configure the site to run locally, and import a database backup. Prerequisites A Linux system (e.g., Ubuntu 22.04; notes for macOS/Windows included). A Git repository with your WordPress code (e.g., https://github.com/yourusername/my-wp-site.git). A database backup file (e.g., my-wp-site.sql). Basic terminal familiarity. Step 1: Install Apache Apache is a web server to host your WordPress site locally. On Ubuntu Update package lists:sudo apt update Install Apache:sudo apt install apache2 Start Apache and enable it to run on boot:sudo systemctl start apache2 sudo systemctl enable apache2 Verify Apache is running: Open a browser and navigate to http://localhost. You should see the Apache default page. Or run: sudo systemctl status apache2. On macOS Use Homebrew to install Apache:brew install httpd Start Apache:brew services start httpd Verify by visiting http://localhost:8080 (macOS Apache uses port 8080 by default). On Windows Download and install XAMPP (includes Apache, MySQL, and PHP) from https://www.apachefriends.org/. Start the XAMPP Control Panel and enable the Apache module. Verify by visiting http://localhost in a browser. Step 2: Install MySQL MySQL is the database server for WordPress. On Ubuntu Install MySQL:sudo apt install mysql-server Start MySQL and enable it:sudo systemctl start mysql sudo systemctl enable mysql Secure the MySQL installation:sudo mysql_secure_installation Follow prompts to set a root password and secure settings. Log in to MySQL to verify:mysql -u root -p Enter the root password and confirm you can access the MySQL prompt. On macOS Install MySQL via Homebrew:brew install mysql Start MySQL:brew services start mysql Set a root password:mysqladmin -u root password 'yourpassword' Verify by logging in:mysql -u root -p On Windows (with XAMPP) MySQL (MariaDB) is included with XAMPP. Start it from the XAMPP Control Panel. Access phpMyAdmin at http://localhost/phpmyadmin to manage databases. Step 3: Install PHP WordPress requires PHP to process dynamic content. On Ubuntu Install PHP and required modules:sudo apt install php php-mysql libapache2-mod-php php-gd php-mbstring php-xml php-curl Verify PHP version:php -v Enable PHP in Apache: Ensure the PHP module is loaded (usually automatic after installation). On macOS Install PHP via Homebrew:brew install php Link PHP to Apache: Edit Apache config (/usr/local/etc/httpd/httpd.conf) to load the PHP module:LoadModule php_module /usr/local/opt/php/lib/httpd/modules/libphp.so Add PHP handling: SetHandler application/x-httpd-php Restart Apache:brew services restart httpd On Windows PHP is included with XAMPP. Ensure the PHP module is enabled in the XAMPP Control Panel. Step 4: Clone the WordPress Repository Install Git if not already installed: Ubuntu: sudo apt install git macOS: brew install git Windows: Download from https://git-scm.com/downloads or use XAMPP’s Git module. Navigate to the web server’s document root: Ubuntu: cd /var/www/html macOS: cd /usr/local/var/www Windows (XAMPP): cd C:\xampp\htdocs Clone the WordPress repository:git clone https://github.com/yourusername/my-wp-site.git Replace https://github.com/yourusername/my-wp-site.git with your repository URL. Set permissions (Ubuntu/macOS):sudo chown -R www-data:www-data /var/www/html/my-wp-site sudo chmod -R 755 /var/www/html/my-wp-site For macOS, replace www-data with _www. Step 5: Configure Apache to Serve the WordPress Site Ubuntu: Create a virtual host configuration:sudo nano /etc/apache2/sites-available/my-wp-site.conf Add the following, adjusting paths as needed: ServerName my-wp-site.local DocumentRoot /var/www/html/my-wp-site AllowOverride All Require all granted ErrorLog ${APACHE_LOG_DIR}/my-wp-site-error.log CustomLog ${APACHE_LOG_DIR}/my-wp-site-access.log combined Enable the site:sudo a2ensite my-wp-site.conf sudo a2enmod rewrite sudo systemctl restart apache2 Add a local domain to /etc/hosts:sudo nano /etc/hosts Add: 127.0.0.1 my-wp-site.local macOS: Edit Apache config (/usr/local/etc/httpd/httpd.conf) to include the virtual host: ServerName my-wp-site.local DocumentRoot /usr/local/var/www/my-wp-site AllowOverride All Require all granted Restart Apache:brew services restart httpd Update /etc/hosts:sudo nano /etc/hosts Add: 127.0.0.1 my-wp-site.local Windows (XAMPP): Edit C:\xampp\apache\conf\extra\httpd-vhosts.conf: ServerName my-wp-site.local DocumentRoot "C:/xampp/htdocs/my-wp-site" Options Indexes FollowSymLinks AllowOverride All Require all granted Update C:\Windows\System32\drivers\etc\hosts (run Notepad as Administrator):127.0.0.1 my-wp-site.local Restart Apache via the XAMPP Control Panel. Step 6: Configure WordPress Copy wp-config-sample.php to wp-config.php:cp /var/www/html/my-wp-site/wp-config-sample.php /var/www/html/my-wp-site/wp-config.php Edit wp-config.php with your database details:define('DB_NAME', 'my_wp_site');
